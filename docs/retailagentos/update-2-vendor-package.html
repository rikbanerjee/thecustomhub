The insight behind the package

Making a store agent-ready isn't one big custom build. Almost all of it is a shared reasoning engine that any catalog can use. The only store-specific piece is a thin adapter mapping that store's products into the engine's model. Validating it on a reference build proved the split holds.

If the hard part is reusable, onboarding the next store shouldn't start from scratch. The eligibility, pricing, stock, and region reasoning — plus the agent manifest and structured-data output — are identical across merchants. What differs is only how each store shapes its own catalog. That's the seam I packaged around.

What I built: the integration package

A prebuilt, deterministic engine (@retailagentos/engine) plus a small, well-documented adapter contract. Drop the engine in, write one adapter for the store's catalog, and the agent surface — manifest, offer evaluation, quotes, checkout hand-off — comes with it.
