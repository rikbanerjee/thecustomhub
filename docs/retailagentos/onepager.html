The problem nobody warns small shops about

Like most independent stores, The CustomHub is a React single-page app — fast and clean for a shopper with a browser. But agentic commerce doesn't drive a browser. An AI agent hits the URL, gets a blank JavaScript shell, and leaves. Ten realistic agent-shopping tasks were run against the live store. Zero completed in a structured, agent-usable way. That's the starting line almost every small business is standing on today.

Before

Unmodified production store
  • No region data anywhere — "ships to Canada?" was unanswerable, a silent no-match.
  • Every product URL served the same empty SPA shell. A non-JS crawler saw nothing.
  • No structured data: zero JSON-LD, no per-product title, no price an agent could read.
  • /.well-known/ucp returned a false-positive HTML 200 instead of a real manifest.
  • Buying worked only by driving the React cart — no agent-callable surface at all.
  • Bulk quotes dead-ended at a freeform human contact form.

After

RetailAgentOS integrated
  • Every product carries shipsTo: ['US','CA'], gated by an explicit region check.
  • Product pages serve schema.org Product + Offer JSON-LD to non-JS clients.
  • A real UCP manifest advertises eligibility, inventory, and pricing capabilities.
  • Agents call find → evaluate_offer → issue_quote → checkout as clean tool calls.
  • Out-of-region and out-of-stock requests are declined with a reason, never a failed checkout.
  • A quote produces a price-locked token that hands off to a genuine Stripe checkout URL.

The proof: find → quote → checkout, no human

This is a real agent tool session against The CustomHub's RetailAgentOS server. Every value — prices, quote ID, the Stripe URL — is genuine output.

> find_product("bengali t-shirt") → 15 matches, all in_stock, $19.99–$29.99 # incl. "Pagla Khabi Ki?" > evaluate_offer(variant, quantity=1, region="CA") { "purchasable": true, "unitPrice": 19.99, "currency": "USD" } > issue_quote(variant, quantity=1, region="CA") { "quoted": true, "quoteId": "q_6b901481_2d59e1", "expiresInSeconds": 600 } > checkout(quoteId="q_6b901481_2d59e1") { "checkedOut": true, "checkoutUrl": "https://checkout.stripe.com/c/pay/cs_test_a1mHe…" }

The checkout call hit the real, already-deployed Stripe Cloud Function — the URL is a genuine test-mode Stripe Checkout session.

And the part I care about most — it says no for the right reason:

> evaluate_offer(variant, quantity=1, region="GB") { "purchasable": false, "reasonCode": "REGION_RESTRICTED", "message": "The CustomHub ships to US/CA only — cannot sell to GB." } > evaluate_offer(out_of_stock_item, region="US") { "purchasable": false, "reasonCode": "OUT_OF_STOCK", "message": "This item is out of stock… request a notification instead." }

The region check short-circuits before Stripe is ever touched. An out-of-region shopper gets a clear decline, not a broken checkout.

Why this matters

Agentic commerce doesn't fail loudly — it fails silently. An agent that can't read a store just moves on to one it can. The gap isn't the catalog or the prices; it's that everything an agent needs — region eligibility, live stock, a canonical price, a callable checkout — lived only inside a browser session. RetailAgentOS turned each of those into something a machine can query and act on, without changing a single pixel for human shoppers.

The CustomHub is a small shop serving NRI shoppers across the US and Canada — exactly the kind of business usually last to get new infrastructure. By implementing the RetailAgentOS spec as-is, it's proof that agentic readiness isn't reserved for retail giants. It's a step any independent store can take today — and one of them just did it first.
